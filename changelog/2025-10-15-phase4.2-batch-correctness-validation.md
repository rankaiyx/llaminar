## Key Implementation Discoveries

### Critical Pattern: Pipeline Factory Registration

**Issue**: Initial implementation failed with "no creator registered for arch='qwen'"

**Root Cause**: PipelineFactory requires explicit registration before use

**Solution**:
```cpp
void SetUp() override {
    // MUST call this before PipelineFactory::instance().create()
    registerQwenPipeline();
    
    // Now factory can create 'qwen' architecture pipelines
    auto pipeline = PipelineFactory::instance().create(config);
}
```

**Lesson**: This is a non-obvious requirement found by examining `test_qwen_pipeline_batch.cpp`. All tests using PipelineFactory must call `registerQwenPipeline()` in SetUp().

### ModelLoader API Pattern

**Correct Pattern** (discovered through iteration):
```cpp
ModelLoader loader;
loader.loadModel(model_path);  // NOT load() or loadWeights()
TransformerLayerConfig base_config = loader.createLayerConfig();
ModelConfig config(base_config, "qwen");

auto pipeline = PipelineFactory::instance().create(config);
auto weights = pipeline->loadWeights(model_path);  // Load through pipeline
```

### Weight Loading Pattern

**Pattern**: Weights are loaded **through the pipeline**, not ModelLoader
```cpp
// Correct:
auto weights = pipeline->loadWeights(model_path);

// Incorrect (doesn't exist):
loader.populateWeights(*weights);
```
