# Phase 4.3: Performance Benchmark Release Build Requirement

**Date:** 2025-10-15  
**Status:** Configuration Fixed - Ready for Release Build Testing  
**Phase:** 4.3 - Batch Performance Measurement

## Summary

Modified batch performance benchmarks to **require Release builds** and suppress excessive debug logging that was causing test failures.

## Problem Identified

Initial test run revealed critical issues:

### 1. Debug Build Performance
- Debug builds are **5-10x slower** than Release due to:
  - No compiler optimizations (-O0 vs -O3)
  - Additional bounds checking
  - Disabled inlining
  - Extra debugging symbols
- Performance measurements in Debug are **completely misleading**

### 2. Excessive Logging
- Test generated **159,320 lines** of DEBUG output for a single batch
- Log verbosity caused premature test termination
- Made it impossible to see actual benchmark results
- Each operator logged detailed execution info at DEBUG level

### 3. Threading Issue (Separate Investigation Needed)
- Operators reporting single-threaded execution despite `OMP_NUM_THREADS=28`
- `configureOpenMPThreading()` calls show `omp_get_max_threads()` returning 1
- Environment variables correctly exported through MPI
- Requires separate investigation - may be OpenMP runtime issue

## Changes Made

### 1. Test Code (`tests/test_batch_performance.cpp`)

**Added Logger Support:**
```cpp
#include "Logger.h"

void SetUp() override {
    // CRITICAL: Set log level to WARN to suppress excessive debug output
    Logger::getInstance().setLogLevel(LogLevel::WARN);

        std::cout << "\n[NOTICE] Performance benchmarks MUST be run in Release build" << std::endl;
        std::cout << "[NOTICE] Debug builds are 5-10x slower\n" << std::endl;
    }
}
```

**Updated Documentation:**
```cpp
/**
 * **CRITICAL REQUIREMENTS:**
 * - MUST be run in Release build: cmake -B build -S . -DCMAKE_BUILD_TYPE=Release
 * - Debug builds are 5-10x slower and produce misleading results
 * - Run via: ./run_batch_performance.sh (handles environment setup)
 */
```

**Removed:**
- OpenMP debug output (not needed for performance measurement)

### 2. Launch Script (`run_batch_performance.sh`)

**Added Build Type Check:**
```bash
# CRITICAL: Check if this is a Release build
if [ -f "build/CMakeCache.txt" ]; then
    BUILD_TYPE=$(grep "CMAKE_BUILD_TYPE:STRING" build/CMakeCache.txt | cut -d= -f2)
    if [ "$BUILD_TYPE" != "Release" ]; then
        echo "⚠ WARNING: Performance benchmarks require Release build!"
        echo "⚠ Debug builds are 5-10x slower - results will be misleading"
        read -p "Continue with Debug build anyway? (y/N) "
        # Exit if user declines
    fi
fi
```

**Added Environment Variables:**
```bash
export LLAMINAR_LOG_LEVEL=WARN           # Suppress debug output
export ADAPTIVE_DISABLE_COSMA=1          # Baseline measurement
```

## How to Use

### Configure Release Build

```bash
# From workspace root
cmake -B build -S . -DCMAKE_BUILD_TYPE=Release \
  -DCOSMA_WITH_PROFILING=OFF \
  -DBUILD_SHARED_LIBS=OFF \
  -DCMAKE_EXPORT_COMPILE_COMMANDS=ON

cmake --build build --parallel
```

### Run Benchmark

```bash
# Use the launch script (recommended)
./run_batch_performance.sh

# Or run specific test filter
./run_batch_performance.sh --filter '*Prefill*'

# Direct execution (not recommended - misses environment setup)
mpirun -np 2 ./build/test_batch_performance
```
