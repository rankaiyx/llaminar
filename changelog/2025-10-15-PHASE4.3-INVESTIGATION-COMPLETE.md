## Root Cause

**File**: `/workspaces/llaminar/src/QwenPipeline.cpp`  
**Lines**: 2090-2142  
**Function**: `QwenPipeline::prefillBatch`

**Problematic Code Structure**:
```cpp
bool QwenPipeline::prefillBatch(...) {
    // Initialize batch state
    ensureBatchState(batch_size);
    
    // THE PROBLEM: Sequential for-loop
    for (int i = 0; i < batch_size; ++i)  // ❌ Sequential iteration
    {
        // Swap in sequence i's state
        n_past_ = n_past_batch_[i];
        k_cache_ = k_cache_batch_[i];
        v_cache_ = v_cache_batch_[i];
        
        // Process single sequence
        bool success = prefill(token_batches[i], ...);  // ❌ One at a time
        
        // Save sequence i's state
        n_past_batch_[i] = n_past_;
        k_cache_batch_[i] = k_cache_;
        v_cache_batch_[i] = v_cache_;
        
        // Extract logits
        batch_logits.push_back(...);
    }
    // ... concatenate results
}
```

**Why It's Sequential**:
1. Loop processes sequences one-by-one
2. No tensor batch dimension (`[batch, seq, hidden]`)
3. State swapping for each iteration
4. Single-sequence `prefill()` called repeatedly
5. No parallel processing of multiple sequences
