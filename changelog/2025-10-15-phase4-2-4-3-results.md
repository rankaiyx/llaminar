### Critical Performance Analysis

#### 1. Sequential Ratio Evidence

```
Prefill [batch= 2, tokens= 8]:  1749.73 ms [seq_ratio=1.06]
Prefill [batch= 4, tokens= 8]:  3501.42 ms [seq_ratio=1.06]
Prefill [batch= 8, tokens= 8]:  6992.02 ms [seq_ratio=1.06]
Prefill [batch=16, tokens= 8]: 14100.89 ms [seq_ratio=1.07]
Prefill [batch=32, tokens= 8]: 28040.52 ms [seq_ratio=1.06]
```

**Interpretation**:
- `seq_ratio ≈ 1.06` means: "Taking 1.06× the time of processing batches sequentially"
- **Expected for parallel**: `seq_ratio << 1.0` (e.g., 0.05× for 22× speedup)
- **Observed**: `seq_ratio ≈ 1.06` confirms **sequential processing**

#### 2. Linear Time Scaling

| Batch Size | Time (ms) | Time per Sequence | Scaling |
|------------|-----------|-------------------|---------|
| 1 | 883 | 883ms | 1.00× |
| 2 | 1,750 | 875ms | 1.98× (linear) |
| 4 | 3,501 | 875ms | 3.96× (linear) |
| 8 | 6,992 | 874ms | 7.92× (linear) |
| 16 | 14,101 | 881ms | 15.97× (linear) |
| 32 | 28,041 | 876ms | 31.75× (linear) |

**Analysis**: Time scales **linearly** with batch size → Sequential execution confirmed

#### 3. Root Cause: Pipeline Sequential Loop

**Location**: `QwenPipeline.cpp` lines 2090-2142 (from Phase 4.3 findings)

**Current Implementation** (Sequential):
```cpp
for (int i = 0; i < batch_size; ++i) {
    prefill(token_batches[i], ...);  // ❌ ONE AT A TIME
}
```

**Expected Implementation** (Parallel):
```cpp
auto batch_tensor = stack_batch(token_batches);  // [batch, seq, hidden]
auto output = prefill_batch(batch_tensor, ...);   // ✅ ALL AT ONCE
```

**Evidence from Logs**:
- Each sequence gets processed independently
- No batch tensor creation observed
- Sequential timing pattern matches exactly

## Next Steps (Phase 5: Pipeline Integration)

### Critical Path Forward

**Priority 1**: Replace Sequential Loop (IMMEDIATE)

**Location**: `src/QwenPipeline.cpp:2090-2142`

**Required Changes**:
```cpp
// CURRENT (Lines 2090-2142):
for (int i = 0; i < batch_size; ++i) {
    // Process each sequence individually
    prefill(token_batches[i], weights, ctx, logits);
}

// TARGET:
// 1. Stack batch tensors
auto batch_tensor = BatchPaddingUtils::stack_batch(token_batches);

// 2. Single prefill call
auto batch_output = prefill_batch_internal(batch_tensor, weights, ctx);

// 3. Extract per-sequence logits
logits = batch_output->slice_batch(batch_size - 1);  // Last sequence
```

**Implementation Steps**:
1. Create `prefill_batch_internal()` helper (processes stacked batch)
2. Modify `prefillBatch()` to use helper instead of loop
3. Same for `decodeBatch()`
4. Verify correctness tests still pass
5. Re-run performance tests

**Expected Outcome**:
- Throughput: 9.13 tok/s → **200+ tok/s** (22× improvement)
- Sequential ratio: 1.06 → **0.045** (parallel processing)
- Batch=32 time: 28,041ms → **1,273ms** (22× faster)
