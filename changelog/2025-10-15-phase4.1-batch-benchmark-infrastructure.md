```cpp
struct BatchBenchmarkMetrics
{
    // Batch configuration
    int batch_size = 0;
    int sequences_completed = 0;

    // Prefill metrics (aggregate across batch)
    int prefill_tokens_total = 0;        // Sum of all sequences' prefill tokens
    double prefill_time_ms = 0.0;        // Wall time for prefillBatch()
    double prefill_throughput = 0.0;     // Total tokens/sec across batch
    double prefill_latency_per_token = 0.0; // Average ms per token

    // Decode metrics (aggregate across batch)
    int decode_tokens_total = 0;         // batch_size × decode_steps
    double decode_time_ms = 0.0;         // Wall time for all decodeBatch() calls
    double decode_throughput = 0.0;      // Total tokens/sec across batch
    double decode_latency_per_token = 0.0; // Average ms per token

    // Total metrics
    double total_time_ms = 0.0;
    double total_throughput = 0.0;       // (prefill + decode tokens) / total time

    // Performance analysis
    double memory_bandwidth_gbps = 0.0;  // Estimated memory bandwidth utilization
    double efficiency_percent = 0.0;     // Actual vs theoretical max bandwidth

    // Model info
    std::string model_path;
    std::string backend;

    void print() const;  // Professional formatted output
};
```

**Key Metrics**:
- **Aggregate throughput**: Total tokens/sec across all sequences in batch
- **Per-token latency**: Average time per token (important for real-time applications)
- **Memory bandwidth**: GB/s estimated from model size × tokens/sec
- **Efficiency**: Percentage of theoretical max bandwidth achieved

#### MPI Coordination
```cpp
// Rank 0 tokenizes and broadcasts
if (rank == 0) {
    std::vector<int> base_tokens = tokenizer.tokenize(params.prompt);
    for (int i = 0; i < batch_size; ++i) {
        token_batches.push_back(base_tokens);
    }
}

// Broadcast batch configuration
MPI_Bcast(&tokens_per_seq, 1, MPI_INT, 0, MPI_COMM_WORLD);

// Broadcast each sequence
for (int i = 0; i < batch_size; ++i) {
    MPI_Bcast(token_batches[i].data(), tokens_per_seq, MPI_INT, 0, MPI_COMM_WORLD);
}
```

#### Prefill Phase
```cpp
auto prefill_start = std::chrono::high_resolution_clock::now();

if (!pipeline.prefillBatch(token_batches, weights, stage_ctx, prefill_logits)) {
    // Handle failure
}

auto prefill_end = std::chrono::high_resolution_clock::now();

metrics.prefill_tokens_total = batch_size * tokens_per_seq;
metrics.prefill_time_ms = duration(prefill_end - prefill_start).count();
metrics.prefill_throughput = (prefill_tokens_total * 1000.0) / prefill_time_ms;
metrics.prefill_latency_per_token = prefill_time_ms / prefill_tokens_total;
```

#### Decode Phase with EOS Handling
```cpp
std::vector<std::vector<int>> generated_tokens(batch_size);
std::vector<bool> seq_finished(batch_size, false);
int active_sequences = batch_size;

for (int step = 0; step < max_new_tokens && active_sequences > 0; ++step) {
    // Greedy sampling for each sequence (rank 0)
    std::vector<int> next_tokens(batch_size, 0);
    
    if (rank == 0) {
        for (int seq = 0; seq < batch_size; ++seq) {
            if (seq_finished[seq]) continue;
            
            // Find argmax logit
            int best_token = find_argmax(logits, seq);
            next_tokens[seq] = best_token;
            
            // Check EOS
            if (best_token == eos_token) {
                seq_finished[seq] = true;
                active_sequences--;
            } else {
                generated_tokens[seq].push_back(best_token);
            }
        }
    }
    
    // Broadcast tokens and finished flags
    MPI_Bcast(next_tokens.data(), batch_size, MPI_INT, 0, MPI_COMM_WORLD);
    MPI_Bcast(finished_flags.data(), batch_size, MPI_INT, 0, MPI_COMM_WORLD);
    
    // Decode batch
    pipeline.decodeBatch(next_tokens, weights, stage_ctx, decode_logits);
}
```

#### Memory Bandwidth Estimation
```cpp
if (model_size_bytes > 0 && metrics.total_time_ms > 0) {
    // Each token processes the model once (rough estimate)
    double total_data_gb = (total_tokens * model_size_bytes) / (1024^3);
    double time_sec = total_time_ms / 1000.0;
    metrics.memory_bandwidth_gbps = total_data_gb / time_sec;

    metrics.efficiency_percent = (memory_bandwidth_gbps / 100.0) * 100.0;
```

## Architecture

### Metric Categories

1. **Configuration**: batch_size, sequences_completed
2. **Prefill**: tokens_total, time_ms, throughput, latency_per_token
3. **Decode**: tokens_total, time_ms, throughput, latency_per_token
4. **Total**: total_time_ms, total_throughput
5. **Performance**: memory_bandwidth_gbps, efficiency_percent

### Calculation Formulas

```
prefill_throughput = prefill_tokens_total * 1000 / prefill_time_ms
prefill_latency_per_token = prefill_time_ms / prefill_tokens_total

decode_throughput = decode_tokens_total * 1000 / decode_time_ms
decode_latency_per_token = decode_time_ms / decode_tokens_total

total_throughput = (prefill_tokens_total + decode_tokens_total) * 1000 / total_time_ms

memory_bandwidth_gbps = (total_tokens * model_size_bytes / 1GB) / (total_time_ms / 1000s)
efficiency_percent = (actual_bandwidth / theoretical_max_bandwidth) * 100
```

### Design Decisions

**Aggregate vs Per-Sequence Metrics**:
- Report **aggregate throughput** (total tok/s across batch)
- This is the primary metric for measuring parallelization benefit
- Example: batch_size=8 @ 160 tok/s aggregate = 20 tok/s per sequence

**Latency Tracking**:
- Also report **per-token latency** (ms/token average)
- Critical for real-time applications (chatbots, voice assistants)
- Lower is better for user experience

**Memory Bandwidth**:
- Estimated from model size × tokens/sec
- Rough approximation (doesn't account for cache, KV cache, etc.)
- Useful for comparing to theoretical hardware limits

**EOS Handling**:
- Track finished sequences individually
- Continue decode until all sequences finish or max_new_tokens
- Prevents wasted computation on finished sequences

## Integration

### Usage Pattern

```cpp
#include "BenchmarkRunner.h"

// Load model
QwenModelWeights weights;
loadModel("model.gguf", weights);

// Create pipeline
auto pipeline = std::make_unique<QwenPipelineAdapter>(config);

// Create tokenizer
auto tokenizer = chat::createTokenizer(loader);

LlaminarParams params;

params.prompt = "Your prompt here";
params.n_predict = 50;

// Run batch benchmark
int batch_size = 8;
size_t model_size_bytes = 500 * 1024 * 1024;  // 500MB

auto metrics = runBatchBenchmark(
    *pipeline, weights, *tokenizer, params, 
    batch_size, model_size_bytes
);

// Print results
metrics.print();

// Access metrics
std::cout << "Aggregate throughput: " << metrics.total_throughput << " tok/s\n";

```

## Metrics Comparison: Batch vs Single

| Metric | Single (BenchmarkMetrics) | Batch (BatchBenchmarkMetrics) |
|--------|---------------------------|-------------------------------|
| Tokens | `prefill_tokens` (one seq) | `prefill_tokens_total` (all seqs) |
| Throughput | Per-sequence tok/s | **Aggregate** tok/s across batch |
| Latency | N/A (implicit) | **Explicit** ms/token average |
| Bandwidth | Not tracked | **Tracked** GB/s with efficiency % |
| Completion | Implicit (1 sequence) | **Explicit** sequences_completed field |

**Key Difference**: Batch metrics emphasize **aggregate throughput** (primary scaling metric) while also tracking **per-token latency** (user experience metric).
