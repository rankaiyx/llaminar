# Inference Parity Test Infrastructure - 2025-10-14

## Summary

Added infrastructure for testing inference parity between Llaminar and PyTorch on real English prompts. The test ensures that both implementations generate identical token sequences when using greedy decoding (temperature 0.0).

## Components Added

### 1. PyTorch Text Generation Script
**File:** `python/reference/generate_text_reference.py`

Generates text using PyTorch/HuggingFace transformers and saves:
- Prompt tokens
- Generated token IDs
- Generated text

**Usage:**
```bash
python3 python/reference/generate_text_reference.py \
  --model models/qwen2.5-0.5b-instruct-q4_0.gguf \
  --prompt "The capital of France is" \
  --max-tokens 20 \
  --temperature 0.0 \
  --output /tmp/reference.json
```

### 2. C++ Inference Parity Test
**File:** `tests/test_inference_parity.cpp`

**Status:** 🚧 **WORK IN PROGRESS** - Infrastructure added, implementation needs completion

Compares Llaminar vs PyTorch text generation:
1. Finds all `.gguf` models in `models/` directory
2. For each model:
   - Generates PyTorch reference (greedy decoding, temp=0.0)
   - Generates Llaminar output with same prompt
   - Compares token sequences - they should match exactly

**Test registered in:** `CMakeLists.txt`
```cmake
add_executable(test_inference_parity tests/test_inference_parity.cpp ...)
add_llaminar_mpi_test(InferenceParityTest 2 test_inference_parity)
```

### 3. Simple JSON Parser
Implemented lightweight JSON parser in test file to avoid external dependencies. Parses only the specific format generated by our Python script.

## Design Rationale

**Why greedy decoding (temp=0.0)?**
- Deterministic - same prompt always produces same output
- No randomness - easier to debug divergence
- PyTorch and Llaminar should produce identical results

**Why test all models?**
- Catches model-specific bugs
- Validates different quantization formats (FP32, FP16, Q4_0, etc.)
- Ensures consistent behavior across model variants

**Why real prompts vs synthetic tokens?**
- Tests full pipeline including tokenization
- More realistic usage pattern
- Catches tokenizer bugs and chat template issues
